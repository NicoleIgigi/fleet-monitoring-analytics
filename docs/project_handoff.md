## File-by-file purpose

### Root
- `README.md`: Project overview, dashboard previews, metric definitions, findings, and pipeline instructions.
- `LICENSE`: MIT license text.
- `requirements.txt`: Python dependencies (pandas, numpy, matplotlib, duckdb, jupyter, pyarrow).

### `data/`
- `data/raw/*.csv`: Raw synthetic datasets (flights, events, maintenance, routes, vehicles) generated by `python/src/generate_data.py`.
- `data/clean/*.csv`: Cleaned datasets (currently identical to raw; cleaning is a warehouse step).
- `data/outputs/weekly_metrics.csv`: Output of `sql/metrics.sql` via DuckDB; used by Power BI.
- `data/outputs/event_type_summary.csv`: Output of `sql/event_type_summary.sql` via DuckDB; used by the drivers dashboard.

### `docs/`
- `docs/metrics_definition.md`: Metric definitions, thresholds, and assumptions.
- `docs/portfolio_story`: Project narrative (problem, users, outputs).
- `docs/project_handoff.md`: This document.

### `sql/`
- `sql/metrics.sql`: Reads CSVs with DuckDB, builds weekly metrics by region/model, computes reliability, critical events per 100 flights, and WoW change.
- `sql/event_type_summary.sql`: Summarizes event drivers by type (total vs critical events) and rate per 100 flights.

### `snowflake/`
- `snowflake/01_setup.sql`: Creates database and RAW/CLEAN/ANALYTICS schemas.
- `snowflake/02_stage_load.sql`: Example RAW table definition + optional staging/COPY steps.
- `snowflake/03_marts.sql`: Example analytics view for weekly metrics; placeholder for event-type mart.
- `snowflake/README.md`: Explains the warehouse layer intent and run order.

### `python/src/`
- `generate_data.py`: Generates reproducible synthetic fleet datasets into `data/raw` and `data/clean`.
- `run_sql_duckdb.py`: Executes `sql/metrics.sql` in DuckDB and writes `data/outputs/weekly_metrics.csv`.
- `run_event_summary_duckdb.py`: Executes `sql/event_type_summary.sql` in DuckDB and writes `data/outputs/event_type_summary.csv`.
- `validate_outputs.py`: Basic data quality checks (required columns, non-negative values, bounds).
- `run_all.py`: One-command pipeline: generate data → metrics → event drivers → validate outputs.

### `powerbi/`
- `Fleet_System_Health_Dashboard.pbix`: Power BI dashboard connected to local CSV outputs.
- `Fleet_System_Health_Dashboard_Snowflake.pbix`: Snowflake-targeted version of the dashboard.
- `powerbi/exports/Fleet_System_Health_Dashboard.pdf`: Static export of the dashboard.
- `powerbi/screenshots/overview_dashboard.png`: Overview dashboard screenshot used in README.
- `powerbi/screenshots/drivers_page.png`: Drivers page screenshot used in README.
